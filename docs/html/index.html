Joint Level Control

Introduction

This is the documentation for the TriPed Projects joint level controller. It implements the Hardware Interfaces required by ros_control to interface the joint position and joint space trajectory controllers with the sensors and actuators. The Naming conventions for each joint, motor and sensor are specified in the wiki.

Structure of the Project

The Project is designed as ROS package and is distributed into the following directories:

  • config
  • docs
  • include
  • launch
  • src

config

The project is designed to be flexible in regards to different configurations, for this reason, hardware adresses and other device specific parameters are always read in from config files. This folder contains all necessairy config files for the joint controllers. On startup of the joint controllers, these config files are first set as ROS parameters and then subsequently read out by the approprate ROS nodes.

docs

The docs folder houses the doxygen documentation from which this page is generated. It is not neccessairy for the proper functioning of this package.

include

The include directory contains the c++ header files of the package. This special directory is necessairy for the catkin_make to find the files. The name of the include directory is specified in the CMakeList file.

launch

The launch directory contains all ROS launch files of the package. It contains the following files

launch file name purpose
hall_sensor_test Testing the Hall Sensors
motor Testing the Motor
joint_level_control Start the Joint controllers

Each launchfile also loads the parameters of the config directory.

src

This directory contains the source code of the package.
